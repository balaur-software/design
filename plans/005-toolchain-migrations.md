## Why this matters

Two forward-compat items with mechanical fixes. (1) `biome.json` uses the
deprecated `recommended` rules field; every `bun run lint` prints a
DEPRECATED info, and the config hard-breaks on Biome 3.x — the vendor ships a
one-command migration. (2) `vite` is pinned `^6` (resolved 6.4.3) while every
installed peer (`@storybook/builder-vite@10.4.6`, `@storybook/react-vite@10.4.6`,
`vitest@4.1.10`, `@vitejs/plugin-react@4.7.0`) already accepts Vite 7 — a
drop-in major that clears one of two majors of lag on the dev/test toolchain
with zero coupled changes. (Vite 8 additionally requires
`@vitejs/plugin-react` ^6 — deliberately deferred, see maintenance notes.)

## Steps

### Step 1: Migrate the Biome config

```bash
bunx biome migrate --write
```

Inspect `git diff biome.json`: the change should be confined to the rules
shape (`recommended` → the new `preset`-style field) plus possibly a schema
version bump. The four deliberate rule settings must survive:
`style/noNonNullAssertion: off`, `suspicious/noArrayIndexKey: off`,
`correctness/useExhaustiveDependencies: off`, the two `a11y` entries.

**Verify**: `bun run lint` → exit 0 AND no `DEPRECATED` text in the output
(`bun run lint 2>&1 | grep -c DEPRECATED` → `0`). `bun run check` → exit 0
with the same result as before the migration (same file count checked, 0
errors, 0 warnings).

### Step 2: Bump Vite to ^7

In `packages/ui/package.json`, change `"vite": "^6"` to `"vite": "^7"`, then:

```bash
bun install
```

**Verify**: `grep -A2 '"vite@' bun.lock | head -3` shows a resolved `vite@7.x`;
`bun install` exited 0 with no peer-dependency warnings mentioning vite.

### Step 3: Exercise everything that touches Vite

```bash
bun run check
bunx playwright install chromium   # no-op if present
bun run --filter '@balaur/ui' test-storybook
bun run build-storybook
```

**Verify**: all exit 0; the storybook test summary shows
`Test Files  119 passed (119)`; `packages/ui/storybook-static/index.html`
exists after the build.
